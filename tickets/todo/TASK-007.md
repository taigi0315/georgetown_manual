# TASK-007: 카드 인스턴트 검색(Instant Search) 추가

## 1. 개요
- 카드가 80개에 달하지만 검색 기능이 없어 특정 장소("H Mart", "Ninja Nation" 등)를 빠르게 찾기 어렵습니다.
- 메인 페이지(`index.html`) 상단에 클라이언트 사이드 즉시 검색창을 추가하여, 입력과 동시에 매칭되는 카드만 필터링하고 자동으로 펼쳐 보여줍니다.

## 2. 세부 작업 항목
- **검색 입력 UI**:
  - 카테고리 필터 바 위(또는 마스트헤드 하단)에 검색 입력 필드 배치. Apple 스타일의 미니멀 디자인, 좌측 돋보기 아이콘 포함.
  - placeholder: "Search places, activities, food…"
  - 우측에 입력 내용을 지우는 클리어(✕) 버튼 표시(입력 시에만 노출).
- **검색 로직 (Vanilla JS)**:
  - 검색 대상: 카드 제목(`.card-title h3`), 한 줄 설명(`.card-brief`), 상세 설명(`.card-desc`), 태그(`.tag`) 텍스트.
  - 대소문자 무시, 공백 trim, 부분 일치(substring) 매칭.
  - 매칭된 카드만 노출하고 `<details>`를 자동으로 `open` 처리. 비매칭 카드는 숨김.
  - 매칭 카드가 0개인 섹션 헤더는 숨기고, 전체 결과 0건이면 "No results" 안내 표시.
  - 검색어가 비면 모든 카드/섹션 원복 및 `open` 상태 해제.
- **카테고리 필터와의 상호작용**:
  - 검색과 카테고리 필터를 AND 조건으로 결합(선택 카테고리 내에서 검색).
- **접근성/성능**:
  - `input` 이벤트에 가벼운 debounce 적용. 키보드 포커스/ESC로 클리어 지원.

## 3. 일정 및 상태
- 상태: ✅ 완료 (2026-06-23, v3.0)
- 우선순위: P1
- 담당자: Antigravity AI
- 구현 요약: `index.html` 마스트헤드/카테고리 바 하단에 검색창 추가. 카드 title/brief/desc/tag 텍스트 부분일치(대소문자 무시), 매칭 시 자동 펼침, 빈 섹션 숨김, no-results 안내, ESC/클리어 버튼 지원. TASK-008과 공유하는 `applyFilters()` 통합 파이프라인으로 카테고리·태그와 AND 결합.
