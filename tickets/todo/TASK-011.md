# TASK-011: 최종 확인일 메타 + 오래된 카드 배지

## 1. 개요
- 가격·영업시간·주소는 시간이 지나면 변하지만, 각 카드 정보가 마지막으로 검증된 시점을 알 수 없습니다.
- 카드에 "최종 확인일"을 기록하고, 일정 기간(예: 6개월)이 지나면 "정보 확인 필요" 배지를 자동 표시하여 신선도를 관리합니다.

## 2. 세부 작업 항목
- **데이터 속성 추가**:
  - 각 카드 `<details>`에 `data-verified="YYYY-MM"` 속성 부여. 1차 일괄 적용 시 현재 릴리스 월(`2026-06`) 기준으로 시드.
- **오래된 카드 배지 (JS)**:
  - 페이지 로드 시 `data-verified`와 현재 날짜를 비교, 6개월 초과 시 카드 요약/상세에 "⚠︎ 정보 확인 필요" 배지 표시.
  - 임계값(개월)은 상수로 분리해 조정 가능하게.
- **선택: 확인일 노출**:
  - 상세 메타 영역에 "Last verified: 2026-06"을 작은 텍스트로 표기.
- **에이전트 SOP 반영**:
  - `agent.html` SOP에 "카드 생성/수정 시 `data-verified`를 당월로 갱신" 단계 추가.

## 3. 일정 및 상태
- 상태: ✅ 완료 (2026-06-23, v3.2)
- 우선순위: P2
- 담당자: Antigravity AI
- 의존성: TASK-013(cards.json) 도입 시 `verified` 필드로 자연스럽게 흡수 가능.
- 구현 요약: 80개 카드 전체에 `data-verified="2026-06"` 시드. 로드 시 현재월과 비교해 `STALE_MONTHS=6` 초과 시 `.tag-row`에 "⚠︎ Verify info" 배지 주입(title에 최종 확인월 표기). 현재(2026-06) 기준 배지 0개 — 신선도 노이즈 없음. jsdom으로 오래된 카드 1개만 배지/신선 카드 무배지 검증. `agent.html` SOP에 "verified 월 갱신" 단계 + 엔트리 템플릿 `data-verified` 속성 추가.
