# TASK-009: 지도 뷰 / 집에서 거리순 정렬

## 1. 개요
- 모든 카드에 집(Georgetown, TX 78633)에서의 이동시간(`Drive: ~XX min`)이 기재되어 있으나, 정렬이나 시각화가 불가능합니다.
- 주말 나들이 동선을 짤 때 유용하도록 (1) 거리순 정렬 토글과 (2) 선택적으로 지도 핀 뷰를 추가합니다.

## 2. 세부 작업 항목
- **이동시간 데이터화**:
  - 각 카드 `<details>`에 `data-drive-min="XX"` 속성을 부여(파싱 가능한 정수). 기존 `Drive: ~XX min` 텍스트에서 추출하거나 수동 기입.
  - 이동시간 정보가 없는 카드는 정렬 시 맨 뒤로.
- **거리순 정렬 토글**:
  - "Sort by distance" 토글. 활성화 시 현재 노출된 카드들을 `data-drive-min` 오름차순으로 재배열(DOM reorder 또는 flex `order`).
  - 섹션 구분을 유지할지/무시할지 결정: 1차 구현은 섹션 내 정렬, 후속으로 전체 통합 정렬 옵션 검토.
- **지도 뷰 (선택, 2차)**:
  - 각 카드에 `data-lat`/`data-lng` 또는 주소 기반 지오코딩으로 좌표 확보.
  - Leaflet + OpenStreetMap 타일로 핀 표시(외부 API 키 불필요). 핀 클릭 시 해당 카드로 스크롤.
  - 리스트 뷰 ↔ 지도 뷰 전환 토글.

## 3. 일정 및 상태
- 상태: 🟡 부분 완료 — 거리순 정렬 ✅ (2026-06-23, v3.4) / 지도 뷰 ⏳ 잔존(P3)
- 우선순위: P1 (정렬) / P3 (지도 뷰)
- 담당자: Antigravity AI
- 비고: 지도 뷰는 좌표 데이터 확보 비용이 있으므로 TASK-013(cards.json) 이후 진행이 효율적.
- 구현 요약(정렬): 런타임에 각 카드의 `#icon-drive` 메타에서 이동시간 파싱(`~8 min`, `~1h 30min`, `~3 hours`, 범위 `15–20 min` 처리 → 분 단위). `data-drive-min`·`data-order` 부여. "↕ Nearest first" 토글로 섹션 내 거리 오름차순 재배열(이동시간 없는 20개 카드는 맨 뒤), 재토글 시 원래 순서 복원. jsdom으로 weekends/restaurants/schools 오름차순·null뒤·복원 검증 완료. 80개 수동 편집 없이 텍스트에서 파싱해 자동 동기화.
- 잔존(지도 뷰): 카드별 `lat`/`lng` 또는 주소 지오코딩 + Leaflet/OSM 핀. TASK-013으로 좌표를 데이터화한 뒤 진행 권장.
