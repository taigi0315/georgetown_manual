AI Agent Update Instructions

This page is for AI agents maintaining the Choi Family Manual. It defines the protocol, templates, and rules for keeping the document current and useful.

===== MISSION =====

Mission

Keep the family manual current, useful, and consistent. Add new restaurants, events, kids' activities, services, and weekend ideas as the family discovers them. Update prices, drive times, phone numbers, and website links as they change. Remove or flag anything that closes.

Proactive discovery is expected. Beyond reacting to family requests, the agent should periodically search for:

  • Upcoming seasonal events, festivals, and limited-time experiences within 2 hours of Georgetown
  • New restaurant openings or highly-rated spots the family hasn't tried
  • Surprise recommendations — things the family might not think to search for but would enjoy (e.g., a meteor shower viewing event, a new trampoline park, a pop-up Korean food truck)
  • Registration deadlines for sports seasons, camps, and school programs
===== FAMILY CONTEXT =====

Family Context

The agent must understand who the family is to make good recommendations:

  • Home address: Georgetown, TX 78633 (Woodlake neighborhood)
  • Older Son (older son, age 11): Loves acting/theater, singing, gardening, FNAF/Little Nightmares (horror games), Fortnite, Ninja Warrior
  • Younger Son (younger son, age 8): Loves drums, Fortnite, Ninja Warrior, soccer

===== SOP =====

Standard Operating Procedure for Updates

  1. Identify the category — the new item belongs to exactly one of the content sections. If unsure, ask the user.
  2. Verify the information — search the web for the current address, phone, hours, price, and link. Never invent details. Any new or changed link must pass the automated link-check workflow (lychee) before merging; it also runs weekly and opens an issue if a link or local image breaks.
  3. Compute drive time from Georgetown, TX 78633. Use Google Maps or similar; cite typical (non-rush-hour) times.
  4. Tag the audience using exactly the tag set defined below.
  5. Follow the entry template exactly. Match the collapsible card structure so layout doesn't break.
  6. Stamp the verified date — set data-verified="YYYY-MM" on the card's <details> to the current month whenever you create or re-confirm a card. After 6 months without an update, a "Verify info" badge appears automatically to flag the card for re-checking.
  7. Update the map coordinates — the map view uses a CARD_COORDS object in index.html keyed by each card's position (DOM order). If you add, remove, or reorder cards, those keys shift and pins will be wrong: regenerate/realign CARD_COORDS after any structural change, and geocode new places (Nominatim/OSM or Google Maps). This positional fragility goes away once TASK-013 moves coordinates into per-card data in cards.json.
  8. Update the masthead — bump the version number (1.x for content adds, 2.0 for restructures) and change LAST UPDATED. Also bump the VERSION constant in sw.js so the PWA cache refreshes for installed users.
  9. Log the change in the changelog at the bottom of this page.

===== ENTRY TEMPLATE =====

Entry Template (Collapsible Card)

Every entry in the manual uses a collapsible <details> element. The summary line shows the name, a brief description, and tags. The expanded content shows full details.

<details class="card" data-verified="YYYY-MM">
  <summary class="card-summary">

    <div class="card-summary-info">
      <div class="card-title">
        <h3>[NAME OF PLACE/ACTIVITY]</h3>
        <!-- Add <span class="star">★</span> for top picks -->
      </div>
      <p class="card-brief">[One-line description visible when collapsed]</p>
      <div class="tag-row">
        <span class="tag tag-haneul">Younger Son</span>

      </div>
    </div>
  </summary>
  <div class="card-body">
    <img class="card-hero-img" src="[Image URL]" alt="[Name of Place/Activity]" loading="lazy">
    <div class="card-meta">

For featured entries: Add class="card featured" to the <details> element and include the ★ star span inside the title.
===== STYLE RULES =====

Style Rules

  • Tone: Honest, practical, slightly opinionated. We are not selling anything to ourselves — say if a place is overpriced, has long lines, or is only worth doing once.
  • Length: Descriptions are 2–4 sentences. Keep it scannable.
  • Drive times: Always from [Private Address], in minutes, normal traffic.
  • Cost format: Dollar amount when known ("$45/person"), ranges ("$20–50"), or "Free."
  • Links: Always use the canonical official site (not Yelp/TripAdvisor) when one exists. Always target="_blank".
  • Don't repeat: Check if an item already exists before adding. Update it instead of duplicating.
  • Images: Include a high-quality horizontal landscape image (aspect ratio ~16:7) at the top of card-body using class="card-hero-img". To support easy identification, a class="card-thumbnail-img" (48x48 rounded square) is automatically generated in the card-summary left sidebar referencing the same image source. Note (v3.1): if a card image fails to load, a category-colored inline-SVG placeholder is shown automatically (no broken-image icon ever appears), so layout stays safe even on a dead URL — but you must still verify image URLs are valid; the fallback is a safety net, not a substitute.
  • Language: English only. Wife reads this document.
===== WHAT TO ADD =====

Topics Always Worth Adding

  • Upcoming festivals and one-time events (especially seasonal — holiday markets, summer concert series)
  • New restaurants the family has tried and rates ≥ 4/5
  • Kids' camps (summer, winter break, spring break) — registration dates matter
  • Service provider referrals the family vouches for
  • Medical providers once chosen
  • Weekend trip discoveries within 3 hours
  • Surprise finds: Gaming tournaments, pop-up events, new parks/attractions, seasonal activities (pumpkin patches, Christmas lights, etc.)
  • Interest-specific: Theater auditions (Older Son), drum workshops/events (Younger Son), Fortnite tournaments (both), horror-themed attractions (Older Son)
===== WHAT NOT TO ADD =====

What NOT to Add

  • Random places the family has no reason to visit. This is not a Yelp scrape.
  • Anything sponsored or paid promotion.
  • Restaurants more than 3 hours away (out of scope).
  • Adult-only nightlife the parents won't realistically use.
===== HOW USER ASKS =====

How the User Will Ask

Typical requests:

  • "Add this restaurant we tried last weekend — [name], it was great."
  • "Update Ninja Nation — they raised prices."
  • "Find 3 new weekend trips we haven't been to yet."
  • "Pull this week's family events in Georgetown and Austin."
  • "Add a new section for [topic]."
  • "What's new this month? Surprise us."

For event roundups, add events into the Events section. Mark time-limited events with a note like [Ends: Aug 2026] so they can be cleaned up later.

===== OUTPUT FORMAT =====

Output Format Requirements

  • This document is HTML. Return the full updated HTML file or a precise diff showing exactly which lines change.
  • Do not change the CSS without explicit user permission.
  • Preserve the section structure and numbering.
  • Preserve existing entries unless the user asks for removal.
  • Use the collapsible <details> card pattern for every new entry.
